# 跑一次就好 — 公家機關辦事指南

> 流程、文件、費用、卡關解法一次備齊，不再白跑公家機關。

**線上版：https://anikievil.github.io/cateye/**

## 這是什麼

台灣公家機關辦事的痛點：不知道要跑哪些單位、缺一份文件被退件重跑、櫃檯問「幾份？用途？」答不出來被兩邊踢皮球。這個 app 把這些坑整理成：

- **38 項業務攻略**（9 大分類），37 項已逐項對照官方公告（查證紀錄見 [docs/verification/](docs/verification/)）
- **嚮導**：選你要辦的事 → 盤點文件「有／還沒」→ 產出缺件解法與等待時間
- **🗣 櫃檯會問**：每份文件旁直接寫好份數、用途、規格的標準答案
- **卡關案例**：真實踩坑經歷 → 查到解法 → 變成後人範例
- **踩坑測驗＋受難賓果**：可分享的小遊戲，題目全部出自查證過的內容
- 文件 checklist、到期提醒（.ics 匯出）、找據點、PWA 離線、大字模式

純前端、零依賴、零框架。資料存 localStorage，不收集任何個資。

## 維護 SOP

內容都在 `js/data.js`（業務）與同檔的 `CASES`（案例）、`QUIZ`（測驗）。

```bash
# 1. 改內容（例：更新規費、新增案例）
vim js/data.js

# 2. 升版（同步 index.html 的 ?v=N 與 service worker 快取）
sh tools/bump.sh

# 3. 上線
git add -A && git commit -m "說明改了什麼" && git push
```

push 後 GitHub Pages 約 1–2 分鐘自動更新。

### 內容守則

- 費用、期限、文件經官方來源核對後，該筆加 `verified: "YYYY-MM-DD"`（頁面顯示綠色 ✓）
- 查不到官方依據的**保持「待查證」，不用記憶補**
- 收到回報 → 查證 → 更新內容或把案例從「徵解法中」改 `status: "solved"`
